## 1\. Estruturas Condicionais (Tomada de Decisão)

- **`if` (Se):** Inicia uma condição.

Python

```
if opcao == 1:
    print('Cadastrar')

```
- **`elif` (Senão Se):** Abre uma nova condição caso a anterior seja falsa. É o `else if` do Java.

Python

```
elif opcao == 2:
    print('Listar')

```
- **`else` (Senão):** O "caso contrário". Executa se nenhuma das condições acima for atendida.

Python

```
else:
    finalizar_app()

```

## 2\. Definições e Blocos de Código

- **`def` (Define):** Usado para criar **Funções**. Uma função é um bloco de código que você "guarda" para usar várias vezes.

> _Dica: Diferente de variáveis (que guardam dados), funções guardam **ações**._

Python

```
def finalizar_app():
    print('Encerrando programa...')

```

## 3\. Tratamento de Erros (Blindagem do Código)

- **`try` (Tente):** O Python tenta executar o bloco de código.
- **`except` (Exceção):** Se algo der errado no `try` (ex: o usuário digitar uma letra onde deveria ser um número), o código "pula" para cá em vez de travar o programa.

## 4\. Listas (Armazenamento Dinâmico)

- **`[]`**: Cria uma lista vazia.
- Deve se criar uma variável antes como pode ver no código a seguir:
- **`.append()`**: Comando para **adicionar** um item ao final da lista.

Python

```python
restaurantes = []
restaurantes.append('Sabor Express') #Agora a lista tem um item
```

- `nome = "Enuk"` (Variável: guarda um valor).
- `def cadastrar():` (Função: guarda um procedimento).

## O que são tuplas e listas?

Ambas são coleções que guardam itens em **ordem** e permitem acesso via **índice** (ex: `lista[0]`).

## 2\. Principais Diferenças

|     |     |     |
| --- | --- | --- |
| **Característica** | **Lista \[ \]** | **Tupla ( )** |
| **Mutabilidade** | **Mutável**: Você pode alterar, adicionar (`append`) ou remover itens. | **Imutável**: Depois de criada, você não mexe mais nela. |
| **Performance** | Mais lenta (exige mais memória para gerenciar mudanças). | Mais rápida e leve para o sistema. |
| **Uso Ideal** | Dados que mudam (ex: lista de compras, tarefas do dia). | Dados fixos/constantes (ex: coordenadas GPS, dias da semana). |

* * *

## Analogias para tuplas e listas:

- **Pense na Lista como um Quadro de Avisos:** Você prega coisas, tira, risca e muda o tempo todo. É flexível, mas gasta mais energia para manter organizado.
- **Pense na Tupla como uma Placa de Inauguração:** Uma vez gravada no bronze (ou parênteses), ela não muda. Se você precisar mudar algo, terá que "derreter" e fazer uma placa nova (criar uma nova tupla).

> **Exemplo:** No Python, se você tem um conjunto de dados que **não deve** ser alterado por segurança (para evitar bugs acidentais), use **Tupla**. Se os dados vão crescer ou diminuir conforme o usuário interage, use **Lista**.

## **For ou While: Qual escolher?**

A escolha depende de uma pergunta simples: **Eu sei quantas vezes isso vai repetir?**

#### **1\. Loop `for` (Repetição Determinada)**

- **Quando usar:** Quando você sabe exatamente o número de vezes que quer repetir ou quando quer percorrer cada item de uma lista/tupla.
- **Exemplo:** "Mande um e-mail para esses **10** clientes" ou "Percorra os **7** dias da semana".
- **Limitação:** Se você definir 3 tentativas e o usuário errar 4 vezes, o programa para e segue em frente, mesmo sem o dado correto.

#### **2\. Loop `while` (Repetição Indeterminada)**

- **Quando usar:** Quando você **não sabe** quantas vezes algo vai acontecer. Ele repete **enquanto** uma condição for verdadeira.
- **Exemplo:** "Peça a senha **até** o usuário acertar" ou "Mantenha o sistema ligado **enquanto** a voltagem for maior que zero".
- **Ponto de Atenção:** Você deve garantir que em algum momento a condição mude para `False`, senão o código entra em **loop infinito** (trava o PC).

* * *

## **Comparação no Código (Exemplo do Número Positivo)**

**Com `for` (Limitado):**

Python

```
for _ in range(3): # Só tenta 3 vezes. Se errar a 4ª, o programa quebra.
    numero = int(input("Digite um número positivo: "))
    if numero > 0: break 

```

**Com `while` (Persistente):**

Python

```
numero = -1
while numero <= 0: # Enquanto o número for ruim, ele não sai daqui.
    numero = int(input("Digite um número positivo: "))
# O código só chega aqui quando o usuário FINALMENTE digitar um número positivo.

```

* * *

**Resumo para sua lógica de Eletricista:** > \* O **`for`** é como um temporizador (roda por X segundos e desliga).

- O **`while`** é como uma boia de nível ou um sensor de presença: enquanto houver água (condição verdadeira), a bomba fica ligada. Se a água acabar, ela desliga.
